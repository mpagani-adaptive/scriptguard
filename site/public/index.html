What it actually finds

The issues that only surface at 2 AM.

Every rule exists because it has caused a real outage. Not hypothetical. Not theoretical. These are the patterns that ship through code review, pass QA, and detonate in production.

Critical

Record loads inside loops

Each record.load() costs 10 governance units. Put it in a loop processing 500 sales orders and your scheduled script dies halfway through. Every time.

Critical

Search calls inside loops

Running search.create() per-record instead of batching. It works in sandbox with 50 records. It crashes in production with 12,000.

Critical

API calls inside loops

HTTP requests to external systems inside a record loop. One slow endpoint response and your entire integration run times out.

Warning

Overloaded User Events

Heavy logic shoved into beforeSubmit because "it just runs on save." Until every sales rep waits 8 seconds to save an order.

Warning

Unbounded search results

.run().each() with no result cap. Works fine until the item table grows past 10,000 rows and the script silently stops returning all results.

Warning

Full record saves

Calling record.save() when you only changed one field. Triggers every User Event, every workflow, every integration on that record. Again.

Warning

No idempotency guards

Script re-runs create duplicate records, double-post journals, or fire webhooks twice. "It only happened once" is never true.

Warning

Hardcoded internal IDs

Works in production, breaks in sandbox, breaks again after the next account refresh. ID 47 means nothing without context.

Why this matters

These aren't bugs. They're time bombs.

The script ran fine last time. That doesn't mean it's working. It means you haven't hit the edge case yet.

Governance crashes

Scripts die mid-execution. Half the records process. The other half don't. Nobody gets an alert. Finance reconciles it two weeks later.

Silent data corruption

Missing error handling means failures don't throw. Records save with bad data. Downstream reports look right until someone audits the numbers.

Integration failures

A loop that worked with 200 records chokes at 2,000. The external system times out. Retry logic doesn't exist. Orders stop syncing.

User experience decay

Overloaded User Events add seconds to every save. Users start avoiding the system. Workarounds proliferate. Data quality tanks.
