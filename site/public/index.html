Record Load Inside Loop critical
Evidence
record.load({ type: 'salesorder', id: ids[i] })
Why It Matters
Each record.load() consumes 10 governance units. Inside a loop processing 200+ records, this exhausts the script's unit budget and causes a mid-execution crash.
Recommendation
Use search.lookupFields() for read-only access, or batch with N/query and Map/Reduce.
Unbounded Search Results warning
Evidence
search.create({ type: 'item' }).run().each(...)
Why It Matters
Without result limits, a search returning thousands of rows will time out or hit governance limits in scheduled and user-event contexts.

Built for NetSuite developers

10 built-in rules targeting the most common SuiteScript production failures.

⚠

Governance Analysis

Detects record loads, searches, and API calls inside loops that burn through governance units and crash scripts mid-execution.

⚡

Performance Patterns

Flags unbounded searches, full-record saves, and heavy operations in User Event scripts that slow down the UI for every user.

✓

Reliability Checks

Catches missing error handling, absent idempotency guards, and hardcoded internal IDs that break across environments.

Static AST Analysis

Native VS Code UI

Get started in 60 seconds

1

Download the extension

Grab the latest .vsix file from the GitHub Releases page.

2

Install in VS Code

Open VS Code, go to Extensions (Ctrl+Shift+X), click the ... menu, and select Install from VSIX.

3

Review your SuiteScript

Right-click any SuiteScript file and select SuiteLens: Review Entire File, or select code and choose Review Selected Code.

☆

Optional: Enable AI review

Add your Anthropic API key in Settings → SuiteLens to get AI-enhanced analysis powered by Claude. Static analysis works without it.
